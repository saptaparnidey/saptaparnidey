## 🧠 About Me

<img align="right" width="340" src="https://github-readme-stats.vercel.app/api/top-langs/?username=saptaparnidey&layout=compact&theme=tokyonight&hide_border=true&title_color=8B5CF6&text_color=c9d1d9&bg_color=0d1117"/>

I am a **Senior Software Engineer** with over **10 years of backend engineering experience**, currently at **Jio Platforms Limited (AJIO)** — one of India's largest fashion e-commerce platforms. My work focuses on designing and owning production microservices that handle fulfilment, shipment orchestration, and reverse logistics at national scale.

I have spent my career building systems where correctness and reliability are non-negotiable: order state machines, real-time carrier integrations, event-driven pipelines, and API layers that serve millions of customers daily. I approach backend engineering with both depth and discipline — from designing clean service contracts to tuning Kafka consumer groups under load.

My core stack is **Java + Spring Boot**, with hands-on production experience in **Apache Kafka**, **Elasticsearch**, **Redis**, **MySQL**, and **Tibco**. I have worked across both product companies (AJIO) and consulting (Cognizant), giving me fluency in both greenfield system design and large-scale legacy modernisation.

I actively use **AI-assisted development tools** — Cursor and Claude Code — as part of my day-to-day engineering workflow to accelerate delivery and improve code quality.

**Open To:** Senior Backend Engineering · Microservices & Platform Engineering · Java Ecosystem · Event-Driven Systems

## 🚀 Featured Projects

<details>
<summary><strong>⚙️ &nbsp; Shipment Service — B2C Fulfilment Orchestration &nbsp;|&nbsp; Jio Platforms / AJIO</strong></summary>
<br/>

> End-to-end shipment lifecycle management system powering AJIO's B2C dispatch pipeline, enabling real-time order fulfilment tracking and improving delivery reliability at scale.

<div align="center">

| Attribute | Detail |
|:---|:---|
| **Stack** | Java · Spring Boot · Apache Kafka · MySQL · Spring Web · Maven |
| **Scale** | Millions of B2C shipments per month · Pan-India logistics network · Multi-carrier |
| **Performance** | Real-time shipment status push to carrier systems · Async event-driven dispatch |
| **Impact** | Improved delivery reliability · End-to-end order fulfilment visibility for customers |
| **Repository** | Internal (Jio Platforms Limited) |

</div>

The Shipment Service is the operational backbone of AJIO's B2C order-to-dispatch flow. Designed and developed from the ground up, it enables real-time tracking of shipment lifecycle events and pushes live status updates to carrier systems. The service exposes APIs consumed by warehouse management systems, carrier integrations, and customer-facing tracking surfaces. Built on Spring Boot with an event-driven backbone over Kafka, the architecture ensures decoupled, scalable processing of high-concurrency dispatch events — including flash sale peaks.

<br/>
</details>

<summary><strong>📦 &nbsp; Fulfilment Returns & Management System — Reverse Logistics Platform &nbsp;|&nbsp; AJIO</strong></summary>
<br/>

<div align="center">

| Attribute | Detail |
|:---|:---|
| **Stack** | Java · Spring Boot · Apache Kafka · MySQL · Elasticsearch |
| **Scale** | Hundreds of thousands of return requests monthly · Multi-category · Pan-India |
| **Performance** | Full order state lifecycle management · Async processing · Real-time status tracking |
| **Impact** | Eliminated Sterling licensing costs · Streamlined reverse logistics · Reduced manual intervention |
| **Repository** | Internal (AJIO.com) |

</div>

The FRAMS system governs the entire reverse logistics loop — from customer-initiated return requests through state transitions, quality check, and final resolution. Built as a Spring Boot microservice, it implements an order state management system that replaced the legacy Sterling platform, enabling in-house control of the return journey with full auditability. Kafka powers asynchronous event propagation across downstream services. The migration eliminated third-party Sterling licensing costs while delivering a more maintainable, extensible architecture owned entirely by the AJIO engineering team.

</details>
